What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of an HTML or XML document as a tree of objects, where each node is an element, attribute, text, etc. The DOM allows programming languages (like JavaScript) to interact with and manipulate the content, structure, and style of a web page dynamically.

How do you select an element from the DOM?

To select an element from the DOM, I can use several methods provided by the document object. First, I can use getElementById() to select an element by its unique id. This is useful when I know the specific identifier for the element I want to manipulate.

Alternatively, I can use getElementsByClassName() to select all elements that share a specific class name. This method returns a collection of elements, which is helpful when I want to target multiple elements with the same styling.

For more flexibility, I can use querySelector(), which lets me select the first element that matches a CSS selector. This is particularly useful for selecting elements based on complex criteria.

Lastly, querySelectorAll() can be used to select all elements that match a given CSS selector, returning a static list that I can iterate over.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique where I attach a single event listener to a parent element instead of adding listeners to each child element. This way, when an event occurs on a child, it bubbles up to the parent, allowing me to handle events more efficiently.
